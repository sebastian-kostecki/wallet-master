# Deposits (lokaty) Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Track term deposits (lokaty) as a separate domain from bank accounts — capital movements excluded from budget P&L, interest as normal income, import auto-detection per bank, manual assign from transactions.

**Architecture:** New `Deposits` domain (`deposits`, `deposit_movements`). Capital transactions on real accounts link via `deposit_movements` (`open` / `close_principal`); budget queries exclude these alongside `transfer_id`. Import uses `Support/Imports/DepositRowClassifier` before P&L category resolution; unmatched close rows surface as candidates on transactions index (mirror transfer banner).

**Tech Stack:** Laravel 13, Inertia v2, Vue 3, Pest 4, MySQL.

**Spec:** `.docs/superpowers/specs/2026-06-11-deposits-design.md`  
**Suggested branch:** `feature/deposits` (do not mix with `improvement/accounts-savings-summary`)
